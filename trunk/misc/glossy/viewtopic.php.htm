After your program runs and generates an output file, these
are the things I'll be checking for the GLOSSY POTM.

1. Did the code generate output within the time limit?
2. Does the output file contain one line for each picture in the input and nothing else?
3. Does each line contain exactly four space-delimited numeric fields following the single letter label?
4. Do the corners implied by each output line fall within the workspace?
5. Does each picture meet minimum area requirements?
6. Check the aspect ratio of each picture to insure it is "close" to that of the input picture (being aware of possible rotation and stretching)
7. Check to insure that there is no overlap if all pictures are placed

Compute the score and tiebreaker (low numbers are good):

Score = page area minus the sum of picture areas
Tiebreaker = area of largest picture minus area of smallest picture

where area (in pixels squared) is the product of height times width in pixels
